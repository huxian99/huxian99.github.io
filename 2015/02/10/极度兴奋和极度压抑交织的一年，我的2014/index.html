极度兴奋和极度压抑交织的一年，我的2014

发表于 2015-02-10   |   分类于 other

1月16号加入一家未融资的创业团队，学习并开发公司聚焦产品的android客户端
2月10号是从老家过完年回来第一天上班，但自己清楚的知道deadline是3月25号
4月18号android客户端1.0上线，3月25号只是努力提交了测试，这半个多月修了n多个bug
9月19号因为一些原因，不得不离职，带着短短几个月的android经历开始忐忑的找工作
10月22号加入格瓦拉，参加电影android客户端的学习和开发

###找android工作之前的点滴
12年12月大四时看同学在腾讯的实习经历可以让他找到很多很好的工作，未来的待遇和前途都很让人羡慕时就决定自己也开始学习编程。努力学了java几个月之后在13年3月25号去第一家公司做asp.net开发，当时周围的同学都说学asp.net没什么前途，但我没得选择，因为也面了太多公司，除了外包公司会要我做java开发，就没有互联网公司或者有互联网基因的公司要我做java，当时也面临毕设和签三方的压力，毕竟还有两个月就大学毕业了，asp.net就asp.net吧。进去之后stone(也就是我的leader)人很nice，对我也很好，很热心的教我开发方面的工作，还有godwin和bond几个从携程跳过来的技术实力都很强，而且每每我有问题都很耐心的给我讲解帮我解决问题，几个月下来之后就会放心的交给我一些任务去开发了。因为EF框架简单好用，把所有东西都封装好了，开发人员就需要调用相应的API就可以完成基本的开发工作了，渐渐地我意识到这样下去对我以后的路的担心，毕竟技术不可能这么简单么，要不怎么会有那么多薪水那么高的人呢，和在腾讯的同学(后来去PPTV了，上个月加入了百度)聊天发现我的技术水的一塌糊涂，http头部有什么东西我都不知道，这个时候也大致知道了国内很多web开发都在选用php而不是asp.net觉得这样下去自己的路会越走越窄，于是决定离职重新开始，12月13号裸辞找android的决定不仅让朋友觉得很惊讶就连我自己也没有把握，但没办法为了自己的前途也只能咬牙坚持。

辞了职之后开始找android的工作，很多互联网公司的简历都投过了，包括知名的和刚刚创立的，但面试的机会寥寥无几，因为一个做asp.net的开发人员想直接找android开发是有点扯淡的，很清楚这一点，但也没有别的办法只能硬着头皮看看有没有机会，同时也在面试一些java web方面的工作(实在不想回到asp.net)，终于有一天一个公司回了我邮件，给我出了一道题，并希望我在xx日期前把apk包和源代码发过去让他们看看我有没有做android开发的潜力，激动了好久，因为这是第一封可能会给到我android开发职位的邮件，好好读好题目便着手下载android studio开始开发，题目现在看来很简单，调用豆瓣的API去搜索图书然后给出一个搜索图书的列表，点击每个图书的item进入一个关于这本图书的详细页面，题目现在看来很简单但当时足足憋了我好几天，一点点猜还搭配着百度搜索(不知道Google好用而且也不会翻墙，一个计算机专业的学生大学过的真的好失败)，好在让我回复邮件的截至日期前弄好了，代码写的七七八八的自己都不懂，但是功能实现了，回复了邮件但然后也就没有然后了。

我把这个小app装在了手机里，以便有面试的机会可以有的聊，幸运的是没过几天我的上家公司就联系我有个面试的机会，自然是很激动，但是面试过程中面试官问的几个问题跟android关系不大，反复盯着我写的这个小app看，叫我等了一会，再后面是两个人进来，其中一个是技术负责人Yoker一个是我后来一起开发android的小伙伴若楠(可是个妹子哦)。问了问我activity的生命周期，这个因为来之前也刷过一些android的面试题，虽然都是背的但对这个印象很深刻，后来给我提了一个场景，当用户在用app时来了一个电话问我activity是生命周期的哪个时刻，完全不知道只能猜，我弱弱的回了应该是onPause，可能也会有onStop吧。他们估计也看出我不知道猜着回答的也就没问太多，之后聊了聊我之前做asp.net时候碰到问题是怎么解决的，也就一五一十的说了，可能我的诚实加了点分就到了hr面，和hr(也就是后来的知心姐姐)聊了好多，知道公司没有融资，待遇不会很高神马神马的，我当时心里想的是可能会有android的offer了吧，特别激动而且对公司融资什么的完全没概念，最后没有当场给offer告诉我他们会开个会讨论一下给我答复。回到家等的第二天就接到了小伙伴若楠的电话，也是一样出了道题直接把一个界面给我让我写出布局文件。之前只会写一些服务端的逻辑，对界面的布局文件基本没什么概念。但这也可能是我唯一的能做android开发的机会了吧，不管怎么样要想办法解决掉它，于是看一些基本的控件和简单的动画，憋了两天总算也弄出来了(其实只完成了80%左右)发过去后又是等，等到了星期五快下班offer电话打过来了，当时很激动很爽快的答应了，虽然待遇很低很低，但很开心。

###Android的开始
1月16号入职了上家公司，周围的小伙伴都很年轻，基本都是大学刚毕业的，若楠带着我和大家伙每个人都打了招呼，Yoker第一天也和我聊了很多很多关于创业方面他的思考，当时还记得他聊到习主席应该站在什么一种什么高度下思考着这些产业链的，让我收获很多很多，现实的情况我也很清楚，android这边我和若楠两个人搞，她也没什么经验，教不了我什么，Yoker最看重我的学习和快速应用的能力，给了我很多信心，然后基本上就是年会(一次20几个人的聚餐，幸运的是抽奖中了个iPad mini)和准备回家过年了，我回家很早因为买不到火车票，提前几天走能买到9折的机票回家，但还是很心疼，毕竟当时兜是比脸还要干净的，现在也差不多，哈哈。

2月10号是春节过后回来上班的第一天，小伙伴们都互相带了家乡的特产，我带了很多哈尔滨红肠，气氛很融洽，也很开心吃到了好多各地的特色，感觉跟大一刚来到上海读书一样，毕竟都是大学刚刚毕业参加工作的小伙伴，有的还是实习生。下午就开了会确定app的deadline是3月25号，时间不是我们估的，我们也估不来，完全没概念，只是希望我们快速开发然后和其他的一些传统公司合作，app的电商入口卖一些他们的产品。就这样开始了我的持续加班经历，边学边应用，碰到问题及时搜索，之间也买了一本疯狂android讲义，是写给入门的人看的，里面讲的通俗易懂，快速了翻了翻控件部分凭着自己的记忆就这样开始了真正的android开发之旅。很快，问题就出现了，印象最深刻的就是一张图片直接让我们的应用crash了，OutOfMemoryError，搜索了很多办法试过之后真的很泄气，后来无意间在StackOverFlow上有个人用WeakReference这玩意解决了，当时激动坏了，但没高兴太久，WeakReference有可能拿到的bitmap是null的，总不能让用户看不到图片吧。玩淘宝的客户端感觉那么多图片都没有crash感觉自己弱爆了，在后来就搜到了开发者官方文档给出的建议，就是根据ImageView的大小去压缩bitmap的大小，用BitmapFactory.Options这玩意，解决这一个问题花的时间特别长，以至于后面面试的过程中人家问我碰没碰到过OOM，第一反应就是一张图片不做处理crash的情况。渐渐的时间到了3月25号我和小伙伴若楠终于把1.0搞好了，本想可以放松下，天知道一测试bug无限多，尤其是小米，当时内心里真是把小米的工程师问候了个遍，又不能不解决，因为小米的用户量也太大了没人敢忽视它，就这样一直到4月18号发布了1.0。

现实的情况永远都是很残酷的，1.0我们测试了10几个机器，但是到了真实环境，问题出现的就更多了。记得老板的一个好朋友下载试用就crash了，若楠很紧张的打电话问是怎么出的问题，然后思考定位问题，解决问题，就这样一步一步走过来。接下来1.1决定整体换android design风格，因为设计师只有一个，所以1.0版本的UI是iOS的风格，过了一个多月1.1发布，这个版本用户投诉的bug相对少了很多，之后就是1.2，1.3的继续开发了。

9月19号因为一些原因不得不离开，带着几个月android经历的我又重新加入了找工作的队伍中。

###Android面试经历
是骡子是马真的要拉出来溜溜了，也不知道自己真正掌握到什么程度，因为身边也没有人做一个标尺告诉我，所以面试还是多想想策略吧，目标肯定是BAT，虽然知道自己肯定达不到，但有了大方向也就不会很迷茫。当时的策略是先排序1. 面试几家传统行业的公司想转型互联网的；2. 面试一些刚刚融资AB轮的；3. 面试一些成熟的小有名气的CD轮公司；4. 最后尝试BAT。在面试的过程中快速解决掉自己不会不懂的，面了几家1的公司就决定不会在面这种类型的公司了，面试过程基本没什么技术含量，就是聊聊当时碰到问题怎么解决的，也没问些具体的问题就会给offer了，待遇还不少，至少在我看来(大牛会觉得没什么吧)；紧接着是面试2，面试机会也挺多的，但相对难了，印象很深的一个问题是什么是单例模式，这个我在用Volley的时候用过，自然的就回答了整个工程只有一个类实例，减少资源的重复创建，紧接着被问到有没有什么要注意的问题，这下糟糕了，不知道，面试官也很nice的教我如果多个线程来调用getInstance，要用同步否则有可能产生多个类实例，那不就违反了单例模式的设计思想了么。回去一查自己当时是根据官方给的建议写的getInstance是public static synchronized MySingleton getInstance(Context context)，我完全没有想过synchronized是干什么用的，只知道应该这么用，因为它是Google官方给出的建议，后来特地查了查单例知道了双重检查锁定，在以后的面试中还有的面试官让我当场写个单例，就非常非常顺手了。其中有个小插曲，就是在9月25号一个很偶然的机会，在一个社交平台吐司上和一个阿里花名叫残章的小伙伴聊了我的经历，告诉他我写的app，可能他当时也下载了这款app吧，他说他愿意内推我去阿里巴巴面试，打开我的gmail邮箱发现这是真的来自阿里巴巴人力资源部的邮件，简单几句话通知我，残章推荐我去应聘云PC，会在10个工作日处理，最后的落款阿里巴巴集团 人力资源部。收到这封邮件的我有点受宠若惊，但大脑还保持着清醒，赶紧问了这个素未谋面又很够哥们的小伙伴阿里的面试都会问些什么，怎么准备，他很nice很细心的告诉我，只要复习好自己的项目就ok了。我耐心的等这10天，10个工作日，却是那么的漫长，特别期待能有电话是从0571打过来的，哪怕真的有一次面试的经历也好。10天10个工作日过去了，没有0571的电话，一切就那么的结束了。

在这之后的面试中很顺利，只要有面试机会的多数都顺利的拿下了offer，但也卡在了3已经CD轮成熟的互联网公司，只有格瓦拉给了我一次面试的机会，后来才知道楠哥(也就是我的leader)不知道什么原因就是想面试我了，可能这就是缘分吧，想我过来聊一聊，这也是我唯一的一次面试成熟团队的机会，肯定不能错过，一遍又一遍的看了我曾经记录的笔记，当时android大概有五六十篇吧，现在是76篇，都是我碰到问题解决问题的总结和学习android的一些总结，早早的到了楼下，喝了杯咖啡静了静神然后上去，整个面试过程非常愉快，体验了我做的项目，碰到问题解决问题的思路和方案，也聊到了曾经用MAT分析内存的一些经历，包括对uil源代码一些简单的理解，性能调优时看了很多Trinea大神的文章，学着用TraceView和hierarchy viewer等等吧，然后他叫我稍等一下，我想大概就八九不离十了，心理不知道有多高兴。之后不是技术部负责人过来而是hr过来和我聊，当时应该是技术部负责人有些事情吧，和hr愉快的聊完接着和技术部负责人聊了些，最后送我走告诉我回家等消息，差不多面试后的第二天快到下班的时候，接到了hr的电话，确定了offer，剩下的就是安排入职的事情，同时自己在杨浦租的房子也到期了，索性这回直接搬到了公司附近住，方便上下班。

###格瓦拉(Gewara)
格瓦拉的小伙伴都非常非常好，楠哥对我也特别的好，只是我比较笨不会说话，但是心理会记得这些好。梅琥也毫无保留的教给我知识技能和他平时聊天时对一些技术的看法，比方说C++语言最难，但他讨厌的是那些用C++就觉得很牛逼而进行装逼的人，讨厌那些用Linux Mac就非得说windows是渣的人，编程语言只是工具，真正难的在编译原理和算法上，只有那些人才是真正的牛逼。还有其他的小伙伴都非常好包括iOS组的亚姐，小桂等等等等。格瓦拉有年会聚餐(虽然没抽到大奖)，小组聚餐，部门聚餐，哈哈。记得部门聚餐有点喝高了，坚持不让人送，坚持不打车，亮亮搀着我这个不懂事的90后下地铁，怕我不安全路上出什么事，我坚决的拒绝了，一是怕麻烦，二是我觉得我没事，哈哈，喝多的人可能都会觉得自己没事吧，最后我凭借着记忆回到了家，倒头就睡，第二天醒来才发现好几个未接电话都是亮亮大哥打过来的，还在微信群里问我有没有到家，心理真的暖暖的，这些我都会记得。只是嘴比较笨，心理想说的话实在太多太多。。。

###Github
14年上半年在老东家开始学会了用Github，最初是搜索时无意发现了23code.com这个网站，里面收集了很多托管在github上面的开源库，喜欢的不得了，但当时也只拿来用，最简单的方式download zip，而不是git clone，没事的时候就开始一个个看23code.com的这些东西，因为它都有gif图片展示，所以以后在自己项目需要的时候以便能及时拿过来用，避免自己重复造轮子。在老东家做开发的时候，app有时非常卡，要解决这些问题，自然也就想到了性能调优，搜索到了Trinea大神的博客，非常喜欢，有时间一篇一篇的读，虽然有很多还不是很理解，但也会学着尝试理解大神的思想和解决问题的方案，记得在他博客的底部给出了他的群号，很有逼格，是群号中间有一个入门必须回答的问题，说白了不让人直接复制，我当时就觉得这个牛，入群的问题时回答了Stackoverflow，毕竟这个经常用么，进入之后发现里面很多牛人，微博的，当当的等等等等，以至于自己都不敢说太多话，害怕哪个大神过来把我打了，哈哈，开个玩笑。同时记得七月底自己订阅的码农周刊推荐了一篇stormzhang的android学习之路，爱上了他的博客也陆续知道了后面的一系列大神级人物，像csdn的郭霖，罗升阳，任玉刚等等，同时加入了stormzhang的qq群，他鼓励我们多用github，甚至一度让我们用github账号作为群昵称，也就是他我知道了github的重要性，毕竟自己之前也在github上download了许多zip应用到了真实的项目中，其中也包括uil等。日子到了11月份，记得有一天Trinea大神在群里问有没有人有兴趣参加开源库的源代码解析，当时就直接私q了Trinea，告诉他我想试试分析uil，因为当时印象最深的就是被一张图片搞的太惨太惨，后面的版本引入uil帮助我太多了，很想了解里面具体的实现，之前也看过一点点，于是就被拉进了Android开源库解析1群，一点点从学习git，写demo，功能介绍，uml类图，详细设计，总体设计，杂谈，到和小伙伴Grumoon互相check，从中学到了好多好多，真心觉得Grumoon是一个对技术有追求，实力又特别强的小伙伴，看着他分析的Volley被无数次的表扬，很为他骄傲，最后Trinea大神补充了对uil的分析和流程图，感谢小伙伴Grumoon和Trinea大神！

###展望2015

  1. 自己的学习计划，github和blog既然已经有了开始，就会继续坚持；读书方面希望自己读完Java并发编程实战，Java数据结构和算法，Head First设计模式等，并希望能有机会将学习到的知识运用在实际项目里，而不只是空谈。
  2. 希望自己谈场恋爱，有生活而不只是工作。一直觉得技术和产品都是为了更好的生活服务的，脱离了生活而大谈特谈技术产品对生活带来的影响不切实际，应该从体验生活开始去感悟，而不是被日新月异的技术节奏带着走。
  3. 锻炼身体，希望跑步能坚持下去，每天不要太多，坚持就好，让自己的身体一直好用且精神状态饱满。
  4. 多交些朋友，行业内外的朋友都希望多交一些，学习他们对同一件事是怎么思考的，找回当年开朗好玩的自己，多去体验和感悟生活，而不是默默的努力压抑着自己。
