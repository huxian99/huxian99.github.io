Java Annotation注解

发表于 2015-08-30

最近在看Dagger2，里面用到了很多注解，提高了开发效率的同时，也并没有使用发射去过多的消耗性能，所以准备尝试使用这开源框架。在使用Dagger2之前，还是简单的学习总结一下Annotation，因为很多知名的开源库都用到它去简化代码，比如Retrofit，ButterKnife, 还有JavaEE的Spring等等。

注解的概念：

An annotation is a form of syntactic metadata that can be added to Java source code. Classes, methods, variables, parameters and packages may be annotated. Unlike Javadoc tags, Java annotations can be reflective in that they can be embedded in class files generated by the compiler and may be retained by the Java VM to be made retrievable at run-time.[2] It is possible to create meta-annotations out of the existing ones in Java.

JDK中的注解:

@Override: 重写父类的方法, 经常见
@Deprecated: 被废弃的，不推荐使用的
@SuppressWarnings: 去掉编译器的警告

自定义注解

根据需求定义的注解

  1. 使用@interface关键字定义注解；
  2. 成员类型是受限的，合法的类型包括原始类型及String, Class, Annotation, Enumeration;
  3. 成员以没有方法体，没有参数，不抛出异常，没有修饰符进行声明；
  4. 如果注解只有一个成员，则可以取名为value(), 在使用时可以忽略成员名和赋值号( = );
  5. 注解类可以没有成员，没有成员的注解称为标识注解(Dagger2 自定义的ActivityScope);
  6. 可以用default为成员指定一个默认值
