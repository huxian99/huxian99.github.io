设计模式之工厂模式(Factory)

发表于 2015-05-21   |   分类于 设计模式   |  

工厂模式定义：

定义了一个创建对象的接口，但由子类决定要实例化的类是哪一个。工厂方法让类把实例化推迟到子类。

UML类图：

使用场景：

  1. 需要使用工厂替代new的场景(创建对象有较多重复的代码)；
  2. 需要隐藏具体实现，并且使抽象与实现解耦合；
  3. 需要灵活、可扩展的框架，且具体类型不多时。

简单Demo：

1. 定义蛋糕超类(Product) 

public abstract class Cake {

    public String name;

    public void prepare() {
        System.out.println("Preparing " + name);
    }

    public void bake() {
        System.out.println("Bake for 25 minutes");
    };

    public void pack() {
        System.out.println("pack for your " + name);
    }

    public String getName() {
        return name;
    }

}

2. 定义具体蛋糕类(ConcreteProduct) 

public class Tiramisu extends Cake {

    public Tiramisu() {
        this.name = "Tiramisu";
    }

}

public class Mcake extends Cake {

    public Mcake() {
        this.name = "Mcake";
    }

}
...

3. 定义蛋糕店超类(Creator) 

public abstract class CakeStore {

    public Cake orderCake(String type) {
        Cake cake = createCake(type);
        cake.prepare();
        cake.bake();
        cake.pack();
        return cake;
    }

    //工厂方法
    public abstract Cake createCake(String type);

}

4. 定义第一家属于你们具体蛋糕店类，你把它取名叫做童话蛋糕店 

public class TaleCakeStore extends CakeStore {

    public Cake createCake(String type) {
        Cake cake = null;
        if ("Mcake".equals(type)) {
            cake = new Mcake();
        } else if ("Tiramisu".equals(type)) {
            cake = new Tiramisu();
        }
        return cake;
    }

}

开业第一天，来了第一个顾客，点了一份提拉米苏，你不知道有多开心 

public class Client {

    public static void main(String[] args) {
        CakeStore store = new TaleCakeStore();
        Cake cake = store.orderCake("Tiramisu");
        System.out.println("Client ordered a " + cake.getName());
    }

}
