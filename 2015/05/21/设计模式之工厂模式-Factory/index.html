设计模式之工厂模式(Factory)

###工厂模式定义：
定义了一个创建对象的接口，但由子类决定要实例化的类是哪一个。工厂方法让类把实例化推迟到子类。

###UML类图：

###使用场景：

  1. 需要使用工厂替代new的场景(创建对象有较多重复的代码)；
  2. 需要隐藏具体实现，并且使抽象与实现解耦合；
  3. 需要灵活、可扩展的框架，且具体类型不多时。

###简单Demo：
你有一个聪明漂亮又贤惠的女朋友，业余爱好是做一些小甜点，烘培一些小蛋糕给你吃，有Tiramisu口味的，还有Mcake口味的，最新又新学了Mousse口味的，每天都变着法子做你最喜欢吃的蛋糕， 然后依偎在你的怀里，问你好吃不好吃。总之把你哄的团团转，你就会每天拼命地加班写代码，拼命地挣钱来维持这份得来不易的幸福，稳稳的幸福。

3. 定义蛋糕店超类(Creator) 

public abstract class CakeStore {

    public Cake orderCake(String type) {
        Cake cake = createCake(type);
        cake.prepare();
        cake.bake();
        cake.pack();
        return cake;
    }

    //工厂方法
    public abstract Cake createCake(String type);

}

4. 定义第一家属于你们具体蛋糕店类，你把它取名叫做童话蛋糕店 

public class TaleCakeStore extends CakeStore {

    public Cake createCake(String type) {
        Cake cake = null;
        if ("Mcake".equals(type)) {
            cake = new Mcake();
        } else if ("Tiramisu".equals(type)) {
            cake = new Tiramisu();
        }
        return cake;
    }

}

开业第一天，来了第一个顾客，点了一份提拉米苏，你不知道有多开心 

public class Client {

    public static void main(String[] args) {
        CakeStore store = new TaleCakeStore();
        Cake cake = store.orderCake("Tiramisu");
        System.out.println("Client ordered a " + cake.getName());
    }

}
