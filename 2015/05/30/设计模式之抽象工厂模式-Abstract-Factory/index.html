设计模式之抽象工厂模式(Abstract Factory)

发表于 2015-05-30   |   分类于 设计模式

抽象工厂模式定义：

提供一个接口，用于创建相关或依赖对象的家族，而不需要明确指定具体类。

UML类图：

使用场景：

需要创建产品家族和想让制造的相关产品集合起来时，可以使用抽象工厂模式。

简单Demo：

继续我们的蛋糕店，因为蛋糕店的生意太好了，就有了一些加盟店，有一些加盟店打着你蛋糕店的品牌，为了获取更多的利益，于是换了一些比较低端的原材料。你知道后当然无比的气氛，nnd，老子打下来的品牌，让你玷污了，这怎么行！所以你的解决方案是在加盟店的城市建立一个原料工厂，保证加盟店的原材料质量。
1. 原材料工厂： 

public interface CakeIngredientFactory {

    public Flour createFlour();
    public Ingredient createIngredient();

}

2. 根据哈尔滨当地特色，提供原材料： 

public class HarbinCakeIngredientFactory implements CakeIngredientFactory {

    public Flour createFlour() {
        return new FineFlour();
    }

    public Ingredient createIngredient() {
        return new HarbinIngredient();
    }

}

3. 其中的原材料超类及具体实现类： 

/**
* 面粉接口
*
*/
public interface Flour {

    public String toString();

}

/**
* 面粉具体实现类
*
*/
public class FineFlour implements Flour {

    public String toString() {
        return "Fine Flour";
    }

}

/**
* 其他原料接口
*
*/
public interface Ingredient {

    public String toString();

}

/**
* 哈尔滨特色的原材料
*
*/
public class HarbinIngredient implements Ingredient {

    public String toString() {
        return "Harbin Ingredient";
    }

}

4. 有了原料工厂，需要修改我们的蛋糕超类了： 

public abstract class Cake {

    public String name;
    public Flour flour;
    public Ingredient ingredient;

    abstract void prepared();

    public void bake() {
        System.out.println("Bake for 25 minutes");
    }

    public void pack() {
        System.out.println("Place Cake in official box");
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return this.name;
    }

    public String toString() {
        StringBuffer result = new StringBuffer();
        if (name != null) {
            result.append(name);
            result.append("\n");
        }
        if (flour != null) {
            result.append(flour);
            result.append("\n");
        }
        if (ingredient != null) {
            result.append(ingredient);
            result.append("\n");
        }
        return result.toString();
    }

}

5. 接下来是蛋糕实体类: 

public class Tiramisu extends Cake {

    CakeIngredientFactory cakeIngredientFactory;

    public Tiramisu(CakeIngredientFactory cakeIngredientFactory) {
        this.cakeIngredientFactory = cakeIngredientFactory;
    }

    void prepared() {
        System.out.println("Preparing " + name);
        flour = cakeIngredientFactory.createFlour();
        ingredient = cakeIngredientFactory.createIngredient();
    }

}

public class Mcake extends Cake {

    CakeIngredientFactory cakeIngredientFactory;

    public Mcake(CakeIngredientFactory cakeIngredientFactory) {
        this.cakeIngredientFactory = cakeIngredientFactory;
    }

    void prepared() {
        System.out.println("Preparing " + name);
        flour = cakeIngredientFactory.createFlour();
        ingredient = cakeIngredientFactory.createIngredient();
    }

}

6. 忙活完这些，该是时候回到我们的蛋糕店了： 

public abstract class CakeStore {

    public Cake orderCake(String type) {
        Cake cake = createCake(type);
        cake.prepared();
        cake.bake();
        cake.pack();
        return cake;
    }

    protected abstract Cake createCake(String type);

}

7. 具体实现类： 

public class HarbinCakeStore extends CakeStore {

    protected Cake createCake(String type) {
        Cake cake = null;
        CakeIngredientFactory factory = new HarbinCakeIngredientFactory();
        if ("Tiramisu".equals(type)) {
            cake = new Tiramisu(factory);
            cake.setName("Tiramisu");
        } else if ("Mcake".equals(type)) {
            cake = new Mcake(factory);
            cake.setName("Mcake");
        }
        return cake;
    }

}

大功告成，再也不用担心我的品牌被砸了，nnd…
喝杯咖啡，敲几行代码测试一下：
public class Client {

public static void main(String[] args) {
    CakeStore store = new HarbinCakeStore();
    Cake cake = store.orderCake("Tiramisu");
    System.out.println("Huxian order a " + cake);
}

}
